name: issuekit-change
description: 记录和管理 Issue 的需求或方案变更。追踪变更对所有文档的影响。当用户提到需求变更、方案调整时使用。
---

## 用户输入

用户提供变更内容的描述（文字或图片）。

## 概述

本 skill 处理开发过程中的需求变更、技术方案调整等，确保所有受影响的文档一致更新，变更被正确追踪。

## 工作流程

### 第 1 步：定位 Issue 并解析变更

1. 读取 `.issuekit/config.yaml` 中的 `issues_dir` 配置项，获取 Issue 文档存放目录（默认为 `issues`）
2. 定位 issue 目录（从用户输入或当前分支，issue 目录位于 `{issues_dir}/{issue-id}/`）
3. 从用户输入解析变更内容
4. 分类变更类型：
   - **需求变更**：产品需求发生变化
   - **方案调整**：技术方案调整
   - **配置变更**：配置项变化
   - **范围变更**：需求范围扩大/缩小

### 第 2 步：影响分析

1. 阅读相关现有文档
2. 对比变更内容与当前内容，识别差异
3. 评估对下游文档的影响：
   - 需求变更 → 可能影响：技术方案、测试方案、发布文档
   - 方案调整 → 可能影响：测试方案、发布文档
   - 范围变更 → 可能影响：所有文档

### 第 3 步：更新受影响文档

对每个受影响的文档：

1. **原地更新内容**（保持可读性，不使用删除线）
2. **追加变更记录条目**到文档的变更记录表：
   ```
   | v{N} | {日期} | {变更人} | {变更描述} | {原因} |
   ```

### 第 4 步：向用户报告

1. 总结变更内容
2. 列出所有更新的文档及版本号
3. 如果下游文档需要手动更新（如技术方案需要重新设计），提醒用户并建议使用对应的 skill
4. 如果变更显著影响范围，建议重新运行需求文档的质量验证
